# 03 - Game Pillars

> "Every feature added to Project Velir must strengthen at least one Game Pillar. If it weakens a pillar, it should be redesigned or removed."

---

# Purpose

Game Pillars define the core philosophy of Project Velir.

They are the guiding principles used to evaluate every mechanic, UI decision, artwork, audio element, and future feature.

Whenever uncertainty arises during development, these pillars take priority over individual ideas.

---

# Pillar 1 — Accessible Strategy

## Objective

The game should be simple enough for anyone to learn but deep enough to reward strategic thinking.

Players should understand the rules quickly while continuously discovering better strategies.

---

## Design Goals

A new player should understand:

- Movement
- Dice
- Resources
- Winning

within five minutes.

Experienced players should continue improving after hundreds of matches.

---

## Examples

Good

✔ Different resource strategies

✔ Trade-offs

✔ Multiple winning approaches

Bad

✘ Complicated calculations

✘ Hidden rules

✘ Excessive menus

---

# Pillar 2 — Meaningful Decisions

Every turn must contain at least one meaningful decision.

The player should constantly evaluate:

- Attack?
- Defend?
- Save Gold?
- Recruit Army?
- Explore?
- Race for Prestige?

There should never be an obvious correct answer.

---

## Examples

Good

"I could recruit soldiers...

...or save money for Velir Aram."

Bad

"I always recruit because it's objectively better."

---

# Pillar 3 — Historical Identity

Project Velir is inspired by ancient Tamil civilization.

It should feel unmistakably South Indian.

This applies to:

- Architecture
- Symbols
- Music
- Clothing
- UI decoration
- Kingdom names
- Events
- Trade

The game is inspired by history rather than attempting to simulate it.

Gameplay always has priority over historical accuracy.

---

## Avoid

European castles

Knights

Dragons

Magic spells

Fantasy races

Vikings

Samurai

Anything that breaks the historical atmosphere.

---

# Pillar 4 — Dynamic Competition

Players should constantly react to changing situations.

No strategy should remain optimal throughout an entire match.

Examples

A wealthy kingdom becomes a military threat.

A weak kingdom discovers an ancient relic.

A player loses an important battle but gains influence.

Every match should evolve naturally.

---

# Pillar 5 — Replayability

No two games should feel identical.

Replayability comes from:

- Four kingdoms
- AI personalities
- Random events
- Velir Aram discoveries
- Player decisions
- Resource management

Replayability should never rely on excessive randomness.

Player choices create replayability.

---

# Pillar 6 — Player Agency

Players should feel responsible for both victories and defeats.

Luck creates opportunities.

Decisions determine outcomes.

The player should rarely think:

"I lost because of bad luck."

Instead they should think:

"I should have prepared better."

---

# Pillar 7 — Elegant Complexity

Complexity should emerge from combining simple systems.

Example

Simple systems:

- Dice
- Gold
- Army
- Prestige

Together create:

Trade-offs

Planning

Timing

Political pressure

Economic warfare

---

# Pillar 8 — Mobile First

Project Velir is designed specifically for smartphones.

Design decisions should prioritize:

Portrait Mode

Large buttons

Short sessions

Fast loading

Touch controls

One-handed play

Desktop conventions should never reduce mobile usability.

---

# Pillar 9 — Fair Play

Every kingdom follows the same rules.

AI never cheats.

Dice probabilities remain equal.

Resources follow identical rules.

Difficulty should come from better decision-making rather than unfair advantages.

---

# Pillar 10 — AI-Native Development

Project Velir is developed using AI-assisted software engineering.

Every gameplay system should be:

Documented

Modular

Testable

Data-driven

AI-readable

Future contributors should understand the project by reading documentation rather than reverse-engineering code.

---

# Pillar Validation Checklist

Before approving any new feature, answer the following questions.

| Question | Yes/No |
|-----------|--------|
| Does it improve strategy? | |
| Does it create meaningful decisions? | |
| Does it fit the historical theme? | |
| Does it improve replayability? | |
| Is it understandable by new players? | |
| Is it enjoyable on mobile? | |
| Can Codex implement it cleanly? | |
| Can AI opponents understand it? | |

If more than two answers are **No**, the feature should be reconsidered.

---

# Pillar Hierarchy

Not every pillar has equal priority.

Priority Order

1. Gameplay

2. Player Experience

3. Strategy

4. Historical Identity

5. Replayability

6. Visual Quality

7. Technical Elegance

Whenever two pillars conflict, higher-priority pillars take precedence.

---

# Final Principle

Project Velir should always feel like a battle of minds rather than a battle of luck.

Every mechanic should help players create memorable stories through intelligent decisions rather than random outcomes.
